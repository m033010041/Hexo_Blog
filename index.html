Patch/Diff - 軟體工程必備

Mar 21, 2017

Getting Started

記錄一下 Patch 的生成方式以及使用方法

Patch 主要的目的是為了在某個程式內
透過此程式為基礎，對特定功能進行新增
中文稱此為“補丁”，但我還是比較喜歡唸”patch”

因此在我的認知下：

  1. 此程式在沒有補丁時，原本的程式還是可以成功編譯以及運作
  2. Patch 可以滿足需要此功能的用戶，但不一定所有人都適用(若都適用應該就會 merge 進主程式裡面了)

OK 開始正文吧！

透過 diff 指令產生 patch file

linux/unix 系統大部分都可以使用 “diff” 指令來產生 patch file
當然 Git, Subversion, CVS 等軟體也可以產生

我們用 diff 跟 git 為例，指令為

diff

diff <file1> <file2> > <patch-file>
file1 是有新增 patch 的完整程式
file2 則是原本沒有加入 patch 的主程式
等等我們透過範例來詳細說明 patch 檔的內容

Git

git diff > <patch-file>>
或是
git diff <commit-num> > <patch-file>
這兩組的差異只是差在有無填入 commit 編號，沒有的話就是用當時 local 的 HEAD commit 為主
