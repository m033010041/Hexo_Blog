Ｃ語言資料轉型

SYNTAX:

(another type)operand

cast 的轉型方式要看compiler來決定

由於轉型並非只是將記憶體區塊換成某形態的方式讀取
例如 IEEE 754 與 一般的 unsigned/signed 整數型態格式就是不同
正常的signed-integer資料排列為：

1 bit 31 bits
signed data

而IEEE 754 的資料排列格式為：

1 bit 8 bits 23 bits
signed exponent fraction

我們以 “5” 這個數值來做例子

signed-integer expression: 0000_0000_0000_0000_0000_0000_0000_0101

如果直接拿這個直接帶入 IEEE 754 的表示法
我們可以透過運算

$$ {(-1)^S} 1.F 2^{E-127} $$

由於 $2^{E-127} = 2^{0-127} = 2^{-127}$ 非常的小，因此算出來是一個趨近於 0 的數值
因此有時候會在轉型時遇到轉成(單/倍)精度浮點數時發生結果有誤

因此我們可以用簡單的 sprintf 與 sscanf 來解決這個問題
